# Política de commits

## Obrigatório
- 1 commit por onda
- mensagem clara
- handoff/documentação da onda
- sem commit gigante misturando múltiplas fases

## Convenção sugerida
- `wave-01: runtime truthfulness`
- `wave-02: scenario domain and persistence`
- `wave-03: node-edge studio baseline`
- ...
- `final-polish: stabilization and release gate`

## Conteúdo mínimo do handoff por onda
- objetivo da onda
- arquivos alterados
- testes executados
- o que ficou pendente
- classificação provisória de risco
