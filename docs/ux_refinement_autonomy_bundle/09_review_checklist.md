# Review checklist for the autonomous UX cycle

## Studio
- [ ] Can create nodes easily
- [ ] Can create edges easily
- [ ] Properties are understandable
- [ ] Connectivity problems are visible before running
- [ ] Required routes are understandable
- [ ] Only business-facing entities are visible in the main Studio surface
- [ ] Technical internal hubs/intermediate nodes are hidden
- [ ] Direct manipulation on the canvas feels primary

## Runs
- [ ] Queue is readable
- [ ] Statuses are clear
- [ ] Run details are understandable
- [ ] Can distinguish scenario vs run vs revision

## Decision
- [ ] Official candidate is obvious
- [ ] Runner-up is obvious
- [ ] Technical tie is explicit
- [ ] Manual final choice is possible
- [ ] Comparison is readable
- [ ] Infeasibility reasons are clear

## General
- [ ] Main navigation is clean
- [ ] Primary action on each screen is obvious
- [ ] JSON/debug details are secondary, not primary
- [ ] README/docs match the product reality
- [ ] The product no longer feels like an engineering console
- [ ] Final visualizations avoid orphan nodes and technical clutter
