# Achados de UX e problema central

## Diagnóstico

A UX atual já é poderosa, mas ainda é melhor para especialistas que sabem o que estão fazendo do que para um usuário que quer um fluxo limpo.

### Principais problemas
1. Muitas ações diferentes na mesma superfície
2. Exposição excessiva de detalhes técnicos cedo demais
3. Falta de progressão natural do tipo:
   - criar cenário
   - validar estrutura
   - executar
   - acompanhar fila
   - comparar
   - decidir
4. Empate técnico já existe no domínio, mas ainda não está completamente central na experiência
5. Conectividade é um problema frequente e precisa ser tratada com prevenção no studio
6. O usuário precisa entender rapidamente:
   - o que falta para um cenário estar "pronto para rodar"
   - o que está rodando agora
   - o que saiu melhor
   - por que saiu melhor

## Problema central

A aplicação precisa deixar de parecer um "console avançado com UI" e passar a parecer uma ferramenta profissional com fluxo guiado.

## Objetivo de UX

Chegar a uma experiência que seja:

- clara para o primeiro uso
- eficiente para usuários recorrentes
- técnica o suficiente para especialistas
- sem esconder auditoria, mas sem colocá-la no centro da jornada principal
